## Classification Discipline

Future specs, docs, tasks, and release notes must use the PRD classifications exactly:

| Classification | Required use |
|---|---|
| Supported | Public/product path exists and performs the user-visible function. |
| Underdeveloped | Substrate exists, but public workflow, integration, diagnostics, or end-to-end rendering is incomplete. |
| Missing | No product feature exists for the scoped map-rendering capability. |
| Pro-gated | Capability exists only when Pro/native symbol is available. |
| Placeholder/fallback | Code path returns non-renderable or zero-geometry fallback when native path is unavailable. |
| Experimental | API exists but behavior is incomplete, unverified, or not safe to present as production-ready. |
| Unsupported | Product cannot render or validate the feature. |
| Non-goal | Feature belongs to web delivery, general DCC, game-engine tooling, or non-map rendering. |

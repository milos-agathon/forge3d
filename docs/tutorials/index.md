# Tutorials

There are two tracks because forge3d serves two audiences through one viewer +
IPC workflow:

- a GIS-facing workflow built around DEMs, overlays, and cartographic output
- a Python-facing workflow built around notebooks, automation, and scene assets

```{toctree}
:maxdepth: 2

gis-track/index
python-track/index
```

Use the GIS track if you already think in rasters, CRS metadata, and map
products. Use the Python track if you want the smallest path from code to a live
viewer or notebook.

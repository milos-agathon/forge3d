## Deduplicated Findings

Confirmed:

1. Python/native API drift exists for several optional paths:
   - ReSTIR symbols used in wrappers but not exported by `_forge3d`.
   - mesh TBN symbols used in wrappers but not exported by `_forge3d`.
2. There is a wrapper ergonomics mismatch for offscreen/viewer:
   - wrappers probe module-level symbols while implementation is on `Scene`.
3. Feature-gated exposure behavior is real and currently active:
   - `cog_streaming` gate around COG registration (`src/lib.rs:4687`).
   - GPU instancing advanced exports behind `enable-gpu-instancing` (`src/lib.rs:4545`).
   - PROJ functions registered, runtime capability can be false (`src/lib.rs:4741`).
4. Package-root exports are intentionally selective relative to native module breadth.

Corrected:

1. `vector.py`, `buildings.py`, and `path_tracing.py` are not pure Python in ownership terms; they are native-integrated/hybrid modules.
2. Module-count discrepancy is a definition issue, not a true mismatch.

Reframed:

1. “Missing feature exposure” should be split into:
   - missing native export,
   - missing root re-export,
   - missing typing/doc discoverability.
2. “Rust vs Python duplicate implementations” should remain inference-labeled unless full runtime call tracing is done.
